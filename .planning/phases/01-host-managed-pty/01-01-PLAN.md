# Plan 01-01: HOST_MANAGED PTY PoC

## Tasks

### Task 1: Wire CPtyHelper into Package.swift
- Add CPtyHelper as C target
- Add as dependency of SmuxApp

### Task 2: Create PTYManager class
- Wraps smux_forkpty(), master fd read/write, child process lifecycle
- DispatchSource for async master fd reading
- Thread-safe: reads on background queue, dispatches to main for ghostty

### Task 3: Modify GhosttyTerminalView for HOST_MANAGED mode
- New init parameter: `managed: Bool` (default false for backward compat)
- When managed=true:
  - Set cfg.backend = GHOSTTY_SURFACE_IO_BACKEND_HOST_MANAGED
  - Set receive_buffer, receive_resize, receive_userdata callbacks
  - Create PTYManager, start reading master fd
  - Pipe master fd output → ghostty_surface_write_buffer() on main thread
  - receive_buffer callback → write to master fd (keyboard input to shell)
  - receive_resize callback → smux_pty_resize()

### Task 4: Update WorkspaceWindowController to use managed mode for ping-pong panes
- When splitting for ping-pong, create HOST_MANAGED terminal views
- Normal splits remain EXEC mode (no regression)

### Task 5: Cleanup on close
- PTYManager.stop(): close master fd, kill child, invalidate DispatchSource
- GhosttyTerminalView.destroySurface() calls ptyManager.stop()

## Verification
- swift build exits 0
- App launches, EXEC mode terminals work (no regression)
- HOST_MANAGED terminal shows shell prompt and accepts input
