## Claude Code CLI

- **Version:** 2.1.81 (Claude Code) — via `npx @anthropic-ai/claude-code --version`
- **Binary:** Not installed globally (`claude` not in PATH). Running via `npx @anthropic-ai/claude-code` or from the Claude Code app process (observed: `claude --dangerously-skip-permissions` in `ps aux`).
- **Non-interactive flag:** `-p` / `--print` — "Print response and exit (useful for pipes). Note: The workspace trust dialog is skipped when Claude is run with the -p mode."
- **Input format:** `--input-format <format>` — choices: `text` (default), `stream-json` (realtime streaming input). Only works with `--print`.
- **Output format:** `--output-format <format>` — choices: `text` (default), `json` (single result), `stream-json` (realtime streaming). Only works with `--print`.
- **Streaming:** Yes — via `--output-format stream-json`. Additionally `--include-partial-messages` includes partial message chunks as they arrive (requires `--print` and `--output-format=stream-json`).
- **Session continuation:**
  - `-c` / `--continue` — Continue the most recent conversation in the current directory.
  - `-r` / `--resume [value]` — Resume a conversation by session ID, or open interactive picker.
  - `--session-id <uuid>` — Use a specific session ID for the conversation.
  - `--fork-session` — When resuming, create a new session ID instead of reusing the original (use with `--resume` or `--continue`).
  - `--no-session-persistence` — Disable session persistence (only works with `--print`).
- **Permission/autonomy modes:**
  - `--dangerously-skip-permissions` — Bypass all permission checks.
  - `--permission-mode <mode>` — choices: `acceptEdits`, `bypassPermissions`, `default`, `dontAsk`, `plan`, `auto`.
- **Tool control:**
  - `--allowedTools <tools...>` — e.g., `"Bash(git:*) Edit"`
  - `--disallowedTools <tools...>` — deny specific tools.
  - `--tools <tools...>` — specify built-in tool set. Use `""` to disable all, `"default"` for all.
- **System prompt override:**
  - `--system-prompt <prompt>` — Replace default system prompt.
  - `--append-system-prompt <prompt>` — Append to default system prompt.
- **Model selection:** `--model <model>` — alias (`sonnet`, `opus`) or full name (`claude-sonnet-4-6`).
- **Budget cap:** `--max-budget-usd <amount>` — only works with `--print`.
- **Worktree:** `-w` / `--worktree [name]` — Create a new git worktree for the session.
- **Tmux integration:** `--tmux` — Create a tmux session for the worktree (requires `--worktree`).
- **Structured output:** `--json-schema <schema>` — JSON Schema for structured output validation.
- **Additional dirs:** `--add-dir <directories...>` — Allow tool access to extra directories.
- **Minimal mode:** `--bare` — Skips hooks, LSP, plugin sync, attribution, auto-memory, background prefetches, keychain reads, CLAUDE.md auto-discovery.
- **Fallback model:** `--fallback-model <model>` — Automatic fallback when default model overloaded (only with `--print`).

### Validated launch command (non-interactive, streaming)

```bash
claude -p --output-format stream-json --dangerously-skip-permissions "your prompt here"
```

### Validated launch command (non-interactive, plain text)

```bash
claude -p --print "your prompt here"
```

### Validated launch command (bi-directional streaming)

```bash
echo '...' | claude -p --input-format stream-json --output-format stream-json
```

## Codex CLI

- **Version:** 0.116.0 (codex-cli) — via `npx @openai/codex --version`
- **Binary:** Not installed globally (`codex` not in PATH). Available via `npx @openai/codex`.
- **Non-interactive subcommand:** `codex exec` — "Run Codex non-interactively".
  - Prompt can be passed as argument or read from stdin (use `-` for stdin).
  - `--json` — Print events to stdout as JSONL.
  - `--ephemeral` — Run without persisting session files to disk.
  - `-o` / `--output-last-message <FILE>` — Write last agent message to file.
  - `--output-schema <FILE>` — Path to JSON Schema for model's final response shape.
  - `--color <COLOR>` — `always`, `never`, `auto` (default: auto).
  - `--progress-cursor` — Force cursor-based progress updates in exec mode.
  - `--skip-git-repo-check` — Allow running outside a Git repository.
- **Approval mode flag:** `-a` / `--ask-for-approval <APPROVAL_POLICY>`
  - `untrusted` — Only run "trusted" commands without asking.
  - `on-failure` — DEPRECATED. Run all commands, ask on failure.
  - `on-request` — Model decides when to ask.
  - `never` — Never ask for user approval; failures returned to model.
- **Sandbox mode:** `-s` / `--sandbox <SANDBOX_MODE>`
  - `read-only`
  - `workspace-write`
  - `danger-full-access`
- **Full-auto shortcut:** `--full-auto` — Convenience alias for `-a on-request --sandbox workspace-write`.
- **Dangerous bypass:** `--dangerously-bypass-approvals-and-sandbox` — Skip all confirmation prompts and execute without sandboxing.
- **Model selection:** `-m` / `--model <MODEL>`
- **Working directory:** `-C` / `--cd <DIR>`
- **Additional dirs:** `--add-dir <DIR>`
- **Config override:** `-c` / `--config <key=value>` — Override config from `~/.codex/config.toml`.
- **Session resume:** `codex resume` — Resume previous interactive session.
- **Session fork:** `codex fork` — Fork a previous interactive session.
- **Review:** `codex review` / `codex exec review` — Run code review non-interactively.
- **Web search:** `--search` — Enable live web search tool.
- **No alt screen:** `--no-alt-screen` — Disable alternate screen (useful for terminal multiplexers).
- **PTY behavior:** Codex uses a TUI by default in interactive mode. The `exec` subcommand is designed for non-interactive/headless use and outputs JSONL with `--json`.

### Validated launch command (non-interactive, JSONL output)

```bash
codex exec --json -a never -s workspace-write "your prompt here"
```

### Validated launch command (non-interactive, full-auto)

```bash
codex exec --full-auto --json "your prompt here"
```

### Validated launch command (non-interactive, dangerous bypass)

```bash
codex exec --dangerously-bypass-approvals-and-sandbox --json "your prompt here"
```

## PTY (macOS)

- **script command:** Works. `script -q /dev/null sh -c 'echo PTY_OK'` produces `PTY_OK` output (with `^D` prefix from `/dev/null` EOF).
- **macOS variant:** BSD `script` — uses `-q` for quiet mode and `/dev/null` as output file to discard the typescript.
- **portable-pty compatibility notes:**
  - macOS uses the `posix_openpt` / `grantpt` / `unlockpt` API for PTY allocation.
  - The `portable-pty` Rust crate supports macOS natively.
  - The `^D` (0x04) in output is the EOF character echoed back; smux should filter control characters from PTY output.
  - No Linux-specific `/dev/ptmx` concerns on this platform.

## Summary for smux Implementation

| Provider    | Non-interactive mode | Output format     | Autonomy bypass                                 | Session resume        |
|-------------|---------------------|-------------------|-------------------------------------------------|-----------------------|
| Claude Code | `-p` / `--print`    | `stream-json`, `json`, `text` | `--dangerously-skip-permissions` or `--permission-mode` | `--resume`, `--continue`, `--session-id` |
| Codex CLI   | `exec` subcommand   | `--json` (JSONL)  | `-a never` or `--dangerously-bypass-approvals-and-sandbox` | `codex resume`, `codex fork` |

### Key architectural observations for smux

1. **Both CLIs support non-interactive + structured output** — Claude via `--print --output-format stream-json`, Codex via `exec --json`. smux can parse structured events from both.
2. **Both have "dangerous" full-bypass modes** — useful for sandboxed environments where smux manages safety.
3. **Both support model selection** — smux can pass `--model` to both providers.
4. **Session management differs** — Claude uses `--session-id` (UUID-based), Codex uses `resume`/`fork` subcommands. smux needs provider-specific session adapters.
5. **Neither CLI is installed globally** — smux should locate them via `npx` fallback or configurable binary paths.
6. **PTY works on macOS** — `portable-pty` crate can be used for interactive mode wrapping if needed, but `exec`/`--print` modes may eliminate the need for PTY entirely in v0.1.
