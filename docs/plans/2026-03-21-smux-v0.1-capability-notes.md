## Claude Code CLI

- **Version:** 2.1.81 (Claude Code) ‚Äî via `npx @anthropic-ai/claude-code --version`
- **Binary:** Not installed globally (`claude` not in PATH). Running via `npx @anthropic-ai/claude-code` or from the Claude Code app process (observed: `claude --dangerously-skip-permissions` in `ps aux`).
- **Non-interactive flag:** `-p` / `--print` ‚Äî "Print response and exit (useful for pipes). Note: The workspace trust dialog is skipped when Claude is run with the -p mode."
- **Input format:** `--input-format <format>` ‚Äî choices: `text` (default), `stream-json` (realtime streaming input). Only works with `--print`.
- **Output format:** `--output-format <format>` ‚Äî choices: `text` (default), `json` (single result), `stream-json` (realtime streaming). Only works with `--print`.
- **Streaming:** Yes ‚Äî via `--output-format stream-json`. Additionally `--include-partial-messages` includes partial message chunks as they arrive (requires `--print` and `--output-format=stream-json`).
- **Session continuation:**
  - `-c` / `--continue` ‚Äî Continue the most recent conversation in the current directory.
  - `-r` / `--resume [value]` ‚Äî Resume a conversation by session ID, or open interactive picker.
  - `--session-id <uuid>` ‚Äî Use a specific session ID for the conversation.
  - `--fork-session` ‚Äî When resuming, create a new session ID instead of reusing the original (use with `--resume` or `--continue`).
  - `--no-session-persistence` ‚Äî Disable session persistence (only works with `--print`).
- **Permission/autonomy modes:**
  - `--dangerously-skip-permissions` ‚Äî Bypass all permission checks.
  - `--permission-mode <mode>` ‚Äî choices: `acceptEdits`, `bypassPermissions`, `default`, `dontAsk`, `plan`, `auto`.
- **Tool control:**
  - `--allowedTools <tools...>` ‚Äî e.g., `"Bash(git:*) Edit"`
  - `--disallowedTools <tools...>` ‚Äî deny specific tools.
  - `--tools <tools...>` ‚Äî specify built-in tool set. Use `""` to disable all, `"default"` for all.
- **System prompt override:**
  - `--system-prompt <prompt>` ‚Äî Replace default system prompt.
  - `--append-system-prompt <prompt>` ‚Äî Append to default system prompt.
- **Model selection:** `--model <model>` ‚Äî alias (`sonnet`, `opus`) or full name (`claude-sonnet-4-6`).
- **Budget cap:** `--max-budget-usd <amount>` ‚Äî only works with `--print`.
- **Worktree:** `-w` / `--worktree [name]` ‚Äî Create a new git worktree for the session.
- **Tmux integration:** `--tmux` ‚Äî Create a tmux session for the worktree (requires `--worktree`).
- **Structured output:** `--json-schema <schema>` ‚Äî JSON Schema for structured output validation.
- **Additional dirs:** `--add-dir <directories...>` ‚Äî Allow tool access to extra directories.
- **Minimal mode:** `--bare` ‚Äî Skips hooks, LSP, plugin sync, attribution, auto-memory, background prefetches, keychain reads, CLAUDE.md auto-discovery.
- **Fallback model:** `--fallback-model <model>` ‚Äî Automatic fallback when default model overloaded (only with `--print`).

### Validated launch command (non-interactive, plain text)

## Codex CLI

- **Version:** 0.116.0 (codex-cli) ‚Äî via `npx @openai/codex --version`
- **Binary:** Not installed globally (`codex` not in PATH). Available via `npx @openai/codex`.
- **Non-interactive subcommand:** `codex exec` ‚Äî "Run Codex non-interactively".
  - Prompt can be passed as argument or read from stdin (use `-` for stdin).
  - `--json` ‚Äî Print events to stdout as JSONL.
  - `--ephemeral` ‚Äî Run without persisting session files to disk.
  - `-o` / `--output-last-message <FILE>` ‚Äî Write last agent message to file.
  - `--output-schema <FILE>` ‚Äî Path to JSON Schema for model's final response shape.
  - `--color <COLOR>` ‚Äî `always`, `never`, `auto` (default: auto).
  - `--progress-cursor` ‚Äî Force cursor-based progress updates in exec mode.
  - `--skip-git-repo-check` ‚Äî Allow running outside a Git repository.
- **Approval mode flag:** `-a` / `--ask-for-approval <APPROVAL_POLICY>`
  - `untrusted` ‚Äî Only run "trusted" commands without asking.
  - `on-failure` ‚Äî DEPRECATED. Run all commands, ask on failure.
  - `on-request` ‚Äî Model decides when to ask.
  - `never` ‚Äî Never ask for user approval; failures returned to model.
- **Sandbox mode:** `-s` / `--sandbox <SANDBOX_MODE>`
  - `read-only`
  - `workspace-write`
  - `danger-full-access`
- **Full-auto shortcut:** `--full-auto` ‚Äî Convenience alias for `-a on-request --sandbox workspace-write`.
- **Dangerous bypass:** `--dangerously-bypass-approvals-and-sandbox` ‚Äî Skip all confirmation prompts and execute without sandboxing.
- **Model selection:** `-m` / `--model <MODEL>`
- **Working directory:** `-C` / `--cd <DIR>`
- **Additional dirs:** `--add-dir <DIR>`
- **Config override:** `-c` / `--config <key=value>` ‚Äî Override config from `~/.codex/config.toml`.
- **Session resume:** `codex resume` ‚Äî Resume previous interactive session.
- **Session fork:** `codex fork` ‚Äî Fork a previous interactive session.
- **Review:** `codex review` / `codex exec review` ‚Äî Run code review non-interactively.
- **Web search:** `--search` ‚Äî Enable live web search tool.
- **No alt screen:** `--no-alt-screen` ‚Äî Disable alternate screen (useful for terminal multiplexers).
- **PTY behavior:** Codex uses a TUI by default in interactive mode. The `exec` subcommand is designed for non-interactive/headless use and outputs JSONL with `--json`.

## PTY (macOS)

- **script command:** Works. `script -q /dev/null sh -c 'echo PTY_OK'` produces `PTY_OK` output (with `^D` prefix from `/dev/null` EOF).
- **macOS variant:** BSD `script` ‚Äî uses `-q` for quiet mode and `/dev/null` as output file to discard the typescript.
- **portable-pty compatibility notes:**
  - macOS uses the `posix_openpt` / `grantpt` / `unlockpt` API for PTY allocation.
  - The `portable-pty` Rust crate supports macOS natively.
  - The `^D` (0x04) in output is the EOF character echoed back; smux should filter control characters from PTY output.
  - No Linux-specific `/dev/ptmx` concerns on this platform.

## Summary for smux Implementation

| Provider    | Non-interactive mode | Output format     | Autonomy bypass                                 | Session resume        |
|-------------|---------------------|-------------------|-------------------------------------------------|-----------------------|
| Claude Code | `-p` / `--print`    | `stream-json`, `json`, `text` | `--dangerously-skip-permissions` or `--permission-mode` | `--resume`, `--continue`, `--session-id` |
| Codex CLI   | `exec` subcommand   | `--json` (JSONL)  | `-a never` or `--dangerously-bypass-approvals-and-sandbox` | `codex resume`, `codex fork` |

### Key architectural observations for smux

1. **Both CLIs support non-interactive + structured output** ‚Äî Claude via `--print --output-format stream-json`, Codex via `exec --json`. smux can parse structured events from both.
2. **Both have "dangerous" full-bypass modes** ‚Äî useful for sandboxed environments where smux manages safety.
3. **Both support model selection** ‚Äî smux can pass `--model` to both providers.
4. **Session management differs** ‚Äî Claude uses `--session-id` (UUID-based), Codex uses `resume`/`fork` subcommands. smux needs provider-specific session adapters.
5. **Neither CLI is installed globally** ‚Äî smux should locate them via `npx` fallback or configurable binary paths.
6. **PTY works on macOS** ‚Äî `portable-pty` crate can be used for interactive mode wrapping if needed, but `exec`/`--print` modes may eliminate the need for PTY entirely in v0.1.
