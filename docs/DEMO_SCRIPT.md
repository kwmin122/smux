# smux 90-Second Demo Script

> 목표: "AI가 서로 검증하는 터미널" 컨셉을 90초 안에 보여주기

## 준비물
- smux 앱 빌드 (`cargo tauri build --debug`)
- claude CLI 설치 (`claude --version`)
- codex CLI 설치 (`codex --version`)
- 프로젝트 폴더 준비 (빈 폴더 or 간단한 프로젝트)

## 스크립트

### 0:00-0:10 — 오프닝
```
[앱 실행. Welcome 화면 보임]
"smux — AI 에이전트가 서로 검증하는 코딩 터미널."
```

### 0:10-0:20 — 터미널 기본기
```
[폴더 열기 → 터미널 모드]
⌘T (새 탭) → 탭 2개 보임
⌘D (수직 분할) → 2분할
⌘F → 검색 오버레이
"일반 터미널 기능 완비. 탭, 분할, 검색."
```

### 0:20-0:35 — AI 핑퐁 진입
```
[AI PING-PONG 버튼 클릭]
두 터미널 패널 등장. 왼쪽: Planner(Claude), 오른쪽: Verifier(Codex)
가운데: READY 상태 표시
```

### 0:35-0:50 — 아이디어 핑퐁
```
[입력창에 "블로그 자동화 CLI 만들어줘" 입력 → GO]
왼쪽 터미널: Claude가 아이디어 브레인스톰 실행
→ 출력 자동 캡처
오른쪽 터미널: Codex가 아이디어 검토 실행
→ "APPROVED" 감지 → 자동으로 Planning 단계 진행
가운데: IDEATION R1 → PLANNING R1
"Claude가 아이디어를 내면, Codex가 자동으로 검증."
```

### 0:50-1:05 — 계획 핑퐁
```
왼쪽: Claude가 구현 계획 작성
오른쪽: Codex가 계획 리뷰
→ "여기 테스트 전략 부족" (REJECTED)
왼쪽: Claude가 테스트 전략 보강하여 재작성
오른쪽: Codex "APPROVED"
"검증자가 부족한 점을 찾으면, 자동으로 수정 후 재검증."
```

### 1:05-1:20 — 실행 핑퐁
```
가운데: EXECUTION R1
왼쪽: Claude가 실제 코드 작성
오른쪽: Codex가 코드 리뷰
"코드 작성도, 리뷰도 자동. 사용자는 지켜보기만."
[사용자가 오른쪽 터미널에 직접 타이핑 → 개입 가능]
"물론 언제든 직접 개입 가능."
```

### 1:20-1:30 — 클로징
```
"smux. AI가 서로 검증하는 터미널.
 하나가 놓치면, 다른 하나가 잡는다.
 github.com/kwmin122/smux"
```

## 핵심 메시지
1. **두 AI가 서로 검증** — 한쪽이 코딩, 다른 한쪽이 리뷰
2. **완전 자동** — 아이디어 → 계획 → 실행까지 자동 핑퐁
3. **사용자 개입 가능** — 언제든 양쪽 터미널에 직접 타이핑
4. **일반 터미널로도 사용 가능** — 탭, 분할, 검색, 테마

## 바이럴 포인트
- "AI가 AI를 검증하는 터미널"
- Claude가 작성 → Codex가 REJECTED → Claude가 수정 → Codex가 APPROVED 장면
- 이 한 장면이 데모의 핵심
